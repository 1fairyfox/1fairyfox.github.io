### 2026-06-22 — Identity, documentation library, and dropdown navigation (0.2.0)

Reframed the site as a project hub + documentation library in a neutral voice, and
expanded the navigation.

- **Identity:** the owner is **Fairy Fox**, not Twilight — updated across the site
  and notes. (Git author identity is unchanged.)
- **Voice:** recorded a standing rule in `notes/context/principles.md` — neutral
  documentation voice, written about the work rather than in the owner's first
  person, no glorifying/elevation, no puns. Rewrote home, projects, about, and the
  intro post accordingly.
- **Header:** redesigned with a gradient `FF` monogram + wordmark (dropped the
  bold-split text) and a dropdown menu (Projects, Docs) with a mobile hamburger
  and a small progressive-enhancement script (`assets/js/nav.js`).
- **Navigation into projects:** the Projects menu links straight into each
  project's own GitHub Pages documentation site, which is served under the custom
  domain (e.g. `fairyfox.io/pokered-save-editor-2/`).
- **Documentation library:** added a `_docs` collection + `doc` layout with a
  sidebar, a `/docs/` landing, and pages for the ecosystem overview, the shared
  engineering standards, and each project. Registry (`_data/projects.yml`) gained
  `docs`/`notes` URLs; added `_data/doc_categories.yml`.

MINOR bump (a real milestone): `0.1.1 → 0.2.0`.

### 2026-06-22 — Initial project: Jekyll home page + cross-project hub

Created `fairyfox.io` from scratch: a clean, custom (no external theme) Jekyll
site that serves as a personal home page and a hub that ties my repositories
together.

- **Site:** home, projects, blog, and about pages; a first blog post; custom
  responsive CSS (light/dark); RSS feed, SEO tags, and a sitemap via plugins.
- **Deploy:** GitHub Actions builds the site with Bundler + Jekyll and publishes
  to GitHub Pages on every push to `main` (no Pages-gem plugin limits). `CNAME`
  set for the `fairyfox.io` custom domain.
- **Hub:** `hub/` holds the cross-project standards, copy-paste templates, and
  the project registry that other repos pull on demand.
- **Notes:** adopted the living-notes system (status / sessions / changelog /
  context / systems / reference / decisions / plans) used across my projects.
- **Sync model:** documented the pull-only, git-only cross-project model and
  seeded `assets/references/` (git-ignored) with shallow clones of
  `pokered-save-editor-2` and `random-ai-prompt`.

Starting version: `0.1.0`.
